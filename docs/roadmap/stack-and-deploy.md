# Стек и деплой

## Docker

Почему: один VPS, минимум магии, переносимость, быстро обновлять, легко докинуть ffmpeg и (если надо) Postgres.

### Что деплоим через Docker Compose (цель для продакшена)

- `bot-core` (Telegram bot + API + логика)
- `scheduler/worker` (может быть тем же сервисом, но отдельным процессом удобнее)
- `db` (Postgres как контейнер)

### Как работаем на старте (dev-режим)

На этапе активной разработки кода:

- **код бота** запускаем локально (быстрее цикл правок и дебага)
- **зависимости** можно поднимать в Docker (Postgres, позже - n8n)

Когда стабилизируем контуры и появится смысл в воспроизводимом окружении - переносим `bot-core` и `worker` в Compose.
