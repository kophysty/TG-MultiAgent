# Язык: Python vs Node/TS - что выбрать с прицелом на будущее

## Если цель - быстрее собрать умного ассистента - Python

Плюсы:

- проще для STT и self-host (если позже захочешь)
- удобнее для очередей, кронов, утренних сводок
- хороший стек под LLM-оркестрацию (структурированный JSON, инструменты, валидация)

## Если цель - потом легко делать web или extension UI - Node/TS

Плюсы:

- фронт, расширение, сервер в одной экосистеме
- проще realtime или websocket штуки для будущего UI

Но: даже если потом будет расширение или веб-интерфейс, ему всё равно нужен backend API. Он может быть на Python, Chrome extension отлично ходит в REST API.

Выбор под задачу "бот + умность + ffmpeg + Notion": **Python**.
